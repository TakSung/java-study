# 목표

현재 python kata 를 위해 만들어진 문서들을 자바로 포팅하는 과정에 있어 도와줘.

# 문서 작성 가이드 라인

- 최상단에는 문서의 목적을 드러낼것
- 상단에 목차를 작성하여 접근을 용의하게 할것 : grep과 같은 명령어로 쉽게 검색이 가능할수 있도록 할것, 마크다운으로 작성할것을 고려하여 해당 제목에 레퍼런스를 걸것
- 사용자(한국인)가 볼 문서는 한글로 작성
- 기술이나 코드 등 영어로 설명할 것은 영어로 작성
- 에이전트만 볼 내용은 토큰 최적화를 위해서 영어로 작성
- 에이전트만 볼 내용이지만 사용자에게 보여줄 예시 문구는 한국어
- 문체는 구조화 되고 요약적인 형태로 작성
- 꼭 필요한 예시가 아니라면 소소코드 작성은 최소화.
- 로직 설명이 필요한 경우 소스코드 전부 작성하는 것이 아니라 수도코드로 핵심 로직만 작성
- 한 문서에 200라인을 넘지 말것 (너무 길면 가독성이 떨어짐)
- 문서가 너무 길면 분리할것. 레퍼런스를 추가하여 컨택스트를 유지할것

# 작성 문서 종류

어떤 문서 인지 파악 후 작업한다.

- 에이전트 (가 사용할) 프롬프트 인덱스 문서
- 에이전트 (가 사용할) 프롬프트 레퍼런스 문서 (상세 문서)
- 사용자도 같이 읽을 가이드 문서

# 문서 종류별 가이드라인

## 에이전트 프롬프트 인덱스 문서

문서의 종류가 많아지면 문서를 분리하게 됨.
그때 각 문서의 역할을 간략히 설명하여 쉽게 접근 할수 있도록 하는 것이 인덱스 문서의 역할

### 작성 순서

- 해당 문서의 목적
- 문서의 목차 : 개요는 앞으로 작성될 블록의 제목을 레퍼런스 할수 있도록 작성(md 문법 사용하기)
- 각 문서에 대한 간략한 설명 : 해당 문서가 적합한 내용인지 판단할 수 있는 간략한 요약과 어떤 트리거가 있을때 해당 문서를 보면 된다는 사용상황

## 에이전트 프롬프트 레퍼런스 문서

특정 목적에 따라 작성된 정보성 문서.
특정 목적에 대해서 작성된 만큼 자세한 설명을 할것.

### 작성 팁

- 상단에 목차를 작성하여 
- 구조적, 명료성, 요약적, 전문적 으로 작성할것
- 사용예시 1가지를 포함할것
- 문서가 너무 길어질 경우 목적에 따라 분리를 고려할것.
- 어떤 작업을 나타낼경우 순차적으로 해당 작업을 설명할것

### 주의사항

레퍼런스가 순환하지 않도록 주의 할것
